# Complete runtime GroovyDoc interoperability before 1.0

Issue #19 blocks AnnoDocimal 1.0. The issue must first be re-scoped from its Groovy-2-era checklist to current Groovy
3, 4, and 5 behavior. Acceptance covers runtime GroovyDoc recognition by extraction and projection, capture short-
circuiting or duplicate avoidance, and deterministic precedence among canonical and interoperable carriers. The
AnnoDocimal protocol remains canonical as established in ADR 0023.

# Require complete repository-owned documentation for 1.0

Documentation completeness blocks AnnoDocimal 1.0. Repository-owned README and detailed docs, published Javadocs,
release notes, and migration guidance must consistently cover artifact selection, both Gradle plugins, capture modes,
supported APIs, Java/Groovy/Gradle compatibility, documentation protocol behavior, and source-projection semantics. A
0.x-to-1.0 migration guide is required for release-facing changes. Historical changelog backfill is evidence-based but
does not block 1.0. GitHub issue #40 owns this gate.

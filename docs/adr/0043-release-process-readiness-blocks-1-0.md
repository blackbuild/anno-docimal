# Require a rehearsed release process for 1.0

Release-process readiness blocks AnnoDocimal 1.0. The repository must own a documented and rehearsed procedure covering
version and tag derivation, signed Maven Central staging, Gradle Plugin Portal publication, changelog and GitHub Release
synchronization, post-publication verification, and failure recovery. End-to-end automation is optional; repeatability,
evidence, and recovery are required. This decision does not authorize publishing from the architecture session. GitHub
issue #45 owns the runbook and rehearsal gate.

# Pair the 1.0 tracker with a version milestone

AnnoDocimal uses a dedicated GitHub `1.0` milestone for every hard release gate, paired with the 1.0 tracking issue. The
milestone supplies queryable scope and progress; the tracker supplies dependency ordering, rationale, and cross-project
sequencing. Post-1.0 and 2.0 work does not enter the 1.0 milestone merely because it was discussed during the same
architecture session.

# Keep generator technology subordinate to behavior

Issue #25 does not block AnnoDocimal 1.0. ASM, JavaPoet, Palantir JavaPoet, or another implementation choice does not
define the supported projection API. The authorized curation pass closed #25 as solution-first and recorded the rationale
under `.out-of-scope/`; a future issue remains welcome when it demonstrates fidelity, compatibility, or maintenance
value. Confirmed projection validity, determinism, and fidelity requirements remain release gates regardless of
implementation technology.

# Retire the Groovy 2 parameter-name workaround

Issue #22 is obsolete and does not belong in the 1.0 plan. The confirmed compatibility floor starts at Groovy 3, where
parameter metadata is available through the supported compiler configuration, and Jenkins no longer justifies a Groovy
2-specific protocol workaround. Issue #22 was closed with that rationale during the authorized 2026-07-16 curation pass.

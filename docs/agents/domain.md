# Domain documentation

Before architecture, API, or naming work, read:

- the root `CONTEXT.md` for canonical AnnoDocimal vocabulary;
- relevant accepted decisions under `docs/adr/`;
- the current work item's evidence and confirmed decisions.

Use terms as defined in `CONTEXT.md`. If code, issue language, or a proposed change conflicts with the glossary, surface
the conflict instead of silently introducing a synonym.

Record a new ADR only for a confirmed decision that is hard to reverse, surprising without its context, and the result of
a real trade-off. Never turn unconfirmed options or investigation notes into accepted architecture.
